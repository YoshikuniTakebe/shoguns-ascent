# Complete board game rules overhaul (v3)

All four v2 issues have been fixed. Online battle resolution now gates on `allBidsSubmitted` before calling `resolveNextBattle`. Per-player bids merge into the existing `warTacticBids` map via `battle.warTacticBids[playerId] = tacticBids`. The separate `doCleanupSeason` store action is removed — cleanup runs exclusively through `advancePhase` at the 'cleanup' case. Season card purchases are guarded by `trainMandateActive`, which is set `true` only during `executeTrain` and reset `false` at the next `chooseMandateTile` call.

**Watch for:** `resolveNextBattle` does not spread `honorTrack` before calling `gainHonor`, so Seppuku mutates the input state's honor track array in-place (confirmed). Seasonal income (`getSeasonalIncome`) awards harvest reward to all tied-force players in a province rather than applying an honor tiebreaker, creating an inconsistency with how `executeHarvest` resolves ties (confirmed).

**Verdict**: APPROVED — the immutability gap is real but low-impact in practice (Zustand replaces the top-level state reference on every `set()` call, so stale references aren't read). The income tie inconsistency is a minor game-balance nuance, not a crash or correctness failure. Ship it.

## High-level view

The bid-submission flow now takes a `playerId` parameter and merges per-player bids into the battle's `warTacticBids` map, with an `allBidsSubmitted` gate on both server and store that prevents premature resolution. The cleanup lifecycle is a single path: `advancePhase` at the 'cleanup' case calls `cleanupSeason` then transitions to the next season or winter. The `trainMandateActive` flag gates card purchases at both the logic layer (`buySeasonCard` returns early if false) and the UI layer (buy buttons disabled).

One new finding: a shallow-copy omission in `resolveNextBattle`. The function spreads `players`, `provinces`, `activeBattles`, and `log` from the input state but not `honorTrack`. When the Seppuku tactic fires `gainHonor(newState, highestBidder)`, it swaps elements in the original array. In the Zustand flow this is benign — the store immediately replaces state via `set()` — but it violates the immutability contract that the rest of the codebase follows and would break if React's `useSyncExternalStore` ever reads a stale reference.

## Honor track immutability gap in battle resolution

`resolveNextBattle` constructs `newState` by spreading selected fields:

```typescript
const newState: GameState = {
  ...state,
  players: state.players.map((p) => ({ ...p, warProvinceTokens: [...p.warProvinceTokens], hostages: [...p.hostages] })),
  provinces: { ...state.provinces },
  activeBattles: state.activeBattles.map((b) => ({ ...b, warTacticBids: { ...b.warTacticBids } })),
  log: [...state.log],
};
```

`honorTrack` is inherited from `...state` as a reference to the same array. The Seppuku case calls `gainHonor(newState, highestBidder)` which swaps elements in-place:

```typescript
function gainHonor(state: GameState, playerId: string): void {
  const idx = state.honorTrack.indexOf(playerId);
  if (idx < state.honorTrack.length - 1) {
    [state.honorTrack[idx], state.honorTrack[idx + 1]] = [state.honorTrack[idx + 1], state.honorTrack[idx]];
  }
}
```

This mutates both `newState.honorTrack` and the caller's `state.honorTrack` since they're the same object. `executeBetray` correctly avoids this by including `honorTrack: [...state.honorTrack]` in its spread. The fix is one line in `resolveNextBattle`'s initialization.

## Seasonal income vs harvest mandate: tied-force divergence

`getSeasonalIncome` (called during `setupSeason`) determines province control with:

```typescript
const isStrongest = state.players.every((other) => {
  if (other.id === playerId) return true;
  return calculateForce(province, other.id) <= playerForce;
});
```

The `<=` means a player tied with another passes the `isStrongest` check — both tied players receive the province's `harvestReward` during season setup. `executeHarvest` detects ties and applies an honor tiebreaker, awarding the reward to exactly one player. Over a 3-season game, tied provinces pay double income at setup but single income during Harvest mandates.
