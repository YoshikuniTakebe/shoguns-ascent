# UI polish: icons, tooltip shift, toast centering, diorama shadows, card translations

Six visual tweaks bundled into one change. Clan tooltips on the main menu now show HonorIcon and CoinIcon before their respective stats. The tea ceremony tooltip arrow shifts 6px left. The rule-violation toast moves from a viewport-fixed position to an absolutely-positioned center of the map container. Diorama figure shields get a heavier double drop-shadow. The season card modal replaces "Monedas: {coins}" text with a CoinIcon + number. And all season card `effect` strings in `game.ts` were translated from English to Spanish.

**Watch for:** The card translations directly contradict the user requirement "Las cartas siguen en inglés" (confirmed). The toast animation's initial keyframe shifts the element off-center via `translateY(-10px)` stacked on the centering transform, which is correct, but the composite `translate(-50%,-50%) translateY(-10px)` means the entrance animation starts 10px above center — fine visually, but worth confirming it matches intent (possible).

## High-level view

The card-effect translation is the blocking issue: the user explicitly stated the cards should stay in English, yet the diff translates every `effect` field across ~90 cards into Spanish. This needs to be reverted.

The toast repositioning moves from `position:fixed` (viewport-relative, top of screen) to `position:absolute` inside `map-container` (which has `position:relative`), centering it within the map viewport. The keyframe animation was correctly updated to preserve the composite transform origin.

The remaining four changes — icon additions in tooltips and the season modal, the tooltip arrow nudge, and the stronger diorama shadows — are cosmetic CSS/JSX with no behavioral risk.

Confidence: **confirmed** — the diff shows English→Spanish replacement on every effect field, and the user instruction is explicit.

## Toast centering inside map-container

The toast moved from:

```css
position: fixed; top: 20px; left: 50%; transform: translateX(-50%);
```

to:

```css
position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%);
```

The JSX was relocated from a standalone block at the bottom of GameBoard's return into the `map-container` div, which has `position: relative` — necessary for `position: absolute` to anchor correctly. The animation keyframe was updated to use the full `translate(-50%,-50%)` as the base transform so the element doesn't jump when the animation completes.

Previously the toast was viewport-fixed, so it was always visible regardless of map pan state. Now it's inside `map-container` but above the `map-canvas` div that receives the pan transform. If `map-container` ever becomes partially off-screen in some layout, the toast would be clipped or hidden — unlikely given the current flex layout but a behavioral difference from the original fixed positioning.

Confidence: **confirmed** — `map-container` has `position:relative` and the toast sits as a direct child outside the panned canvas.

## SeasonCardsModal coin display

The `t('seasonCardsModal.playerCoins', { coins: ... })` i18n call was replaced with an inline `<CoinIcon size={18} color="#DAA520" />` followed by `<span>{currentPlayer.coins}</span>`. This drops the i18n key entirely — if the game ever needs locale-aware coin labels, this path now shows only an icon + number regardless of locale.
