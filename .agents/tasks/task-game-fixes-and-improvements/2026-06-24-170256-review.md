# Six clans, mandate draft, and visual overhaul for Shogun's Ascent

This change reworks the game's faction system from 5 generic clans to 6 thematically distinct clans with unique mechanical powers, replaces the single-mandate-draw politics phase with a draw-3-pick-1 system (draw-4 for Lotus), adds per-player Tea Ceremony turns with explicit alliance negotiation flow, and layers in three new SVG components for visual identity (map background, clan shields, daimyo portraits). The implementation is hotseat-first with WebSocket server support lagging behind the new mandate flow.

**Watch for:** The WebSocket server still calls `executeMandate` directly without the draw/choose flow, so online mode skips the mandate draft entirely (confirmed). Fox and Koi movement powers are mechanically identical two-hop checks despite different flavor text (confirmed). Dragonfly's honor bonus triggers on tie with any other player at max honor, which may be unintentional (likely).

## High-level view

The clan system now defines six clans with powers that manifest at different game phases: Bonsai gets extra recruits, Turtle retains forces in battle and scores shrine bonuses, Dragonfly earns VP from honor supremacy, Lotus draws an extra mandate tile, and Koi/Fox both get extended movement range. The powers are implemented inline in the relevant game logic functions rather than through a dispatch table, scattering clan-specific behavior across `executeRecruit`, `resolveBattleBids`, `scoreRegions`, and `moveForces`.

The mandate draw system replaces the old "click any mandate" UI with a two-step flow: the active player draws tiles from the deck into `drawnMandates`, picks one to execute, and the remainder return to the top of the deck. This creates information asymmetry since the active player knows what's coming next. The deck has only 12 mandates and no guard against exhaustion, which can deadlock the UI.

Tea Ceremony now tracks a `teaTurnIndex` that cycles each player through a propose/accept turn before advancing to politics. The phase transition uses `advanceTeaTurn` rather than the generic `advancePhase`, giving each player an explicit "End My Tea Turn" action.

The visual layer adds three purely presentational SVG components (`JapanMapBackground`, `ClanShield`, `DaimyoPortrait`) with pointer-events-none and proper z-index layering so they don't interfere with game interaction.

## Online mode mandate desync

The server's MANDATE handler in `server/index.ts` calls `executeMandate(l.gameState, data.payload.mandate, data.playerId)` followed by `advancePlayer`. It never calls `drawThreeMandates` or `chooseMandateFromDrawn`. The client-side flow in `chooseMandateAction` sends `{ type: 'MANDATE', payload: { mandate } }` when online, but the server-side code path doesn't return unused mandates to the deck or set `mandateChoicePhase`. This means:

- Online players never draw tiles; the server just executes whatever mandate the client sends.
- The mandate deck drains at 1 per turn instead of 3 per turn, altering game pacing.
- Lotus's draw-4 advantage has no effect online.

The server needs new message types (`DRAW_MANDATES`, `CHOOSE_MANDATE`) that mirror the client-side two-step flow, or the client needs to send the full draw-choose sequence in a single message that the server validates.

## Fox and Koi movement identity

Both movement powers check two-hop reachability with identical logic:

```typescript
const twoHopReachable = from.adjacentRegions.some(mid => {
  const midRegion = newState.regions[mid];
  return midRegion && midRegion.adjacentRegions.includes(toRegion);
});
```

Fox's description says "move through enemy-occupied regions," which implies it should bypass blockade mechanics or treat occupied intermediate regions as passable when others can't. Currently there is no blockade mechanic, so the Fox power is indistinguishable from Koi's. Both clans pay the same cost for the same benefit.

## Dragonfly honor scoring edge case

In `scoreRegions`:

```typescript
const maxHonor = Math.max(...newState.players.map(p => p.honor));
const honorBonus = clan.id === 'dragonfly' && p.honor >= maxHonor ? 1 : 0;
```

The `>=` means any Dragonfly player tied for max honor gets the bonus. The clan selection UI allows duplicate clan picks in hotseat, so two Dragonfly players could both score extra VP every round. Additionally, Dragonfly starts at 5 honor while others start at 3, so the bonus is nearly guaranteed in early rounds unless someone betray-spends honor.

## Turtle battle retention

In `resolveBattleBids`:

```typescript
const lost = clan.id === 'turtle' ? Math.max(0, cur - 1) : cur;
p.reserveForces += lost;
newState.regions[battle.regionId].forces[pid] = cur - lost;
```

When a Turtle player loses with 1 force: `lost = max(0, 0) = 0`, so `forces[pid] = 1 - 0 = 1`. The Turtle player keeps 1 force in the contested region even after losing the battle. This makes Turtle extremely sticky‚Äîthey can never be fully evicted from a region in a single battle. Combined with the shrine VP bonus, this makes Turtle disproportionately strong at holding shrine regions (Edo, Kansai, Kyushu). The implementation matches "loses 1 fewer force" literally, but the game design likely intends the force to go to reserves, not remain on the board.

## Mandate deck exhaustion

`drawThreeMandates` sets `mandateChoicePhase = true` regardless of whether any mandates were actually drawn. If `mandatesDeck` is empty, `drawnMandates` will be `[]` and the UI shows neither the "Draw Mandates" button (because `mandateChoicePhase` is true) nor any mandate buttons (because `drawnMandates` is empty). The player has no way to proceed. The deck has 12 mandates, `maxMandates` is 3-4 per season, and drawing 3 per turn means the deck empties after 4 turns. With 6 players and 4 mandates per season, the deck runs dry within a single season.
