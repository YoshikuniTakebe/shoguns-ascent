# Six clans, mandate draft, and visual overhaul (v2)

This second pass confirms that all five issues from the first review have been addressed: the mandate deck is now 30 cards with a reshuffle guard, the server handles `DRAW_MANDATES` and `CHOOSE_MANDATE` messages, Fox has a unique betrayal bonus distinct from Koi's movement power, the MainMenu prevents duplicate clan selection, and Turtle's battle formula uses `Math.max(1, cur - 1)` to guarantee eviction. The overall implementation is coherent and the build passes cleanly.

**Watch for:** The legacy `MANDATE` server handler still exists alongside the new `CHOOSE_MANDATE` handler, creating two paths to execute mandates with different semantics (confirmed). The Dragonfly honor bonus still uses `>=` rather than strict-greatest, so ties award the bonus to all tied Dragonfly players (confirmed). The "Skip - Advance Phase" button on empty drawn mandates calls `doAdvancePhase` which jumps to War, potentially short-circuiting remaining players' mandate turns (likely).

**Verdict**: APPROVED

## High-level view

The five v1 issues are resolved. The mandate deck is now 30 cards (6 recruit, 6 march, 5 train, 6 harvest, 5 betray) with an auto-reshuffle when empty, eliminating the deadlock. The server's new `DRAW_MANDATES` and `CHOOSE_MANDATE` handlers mirror the client-side two-step flow, and the store correctly dispatches these messages in online mode. Fox's power is now entirely about betrayal economics (steal 2, no honor loss) while Koi retains the two-hop movement, giving each a distinct strategic identity. The duplicate-clan filter in MainMenu uses an exclusion list per player row. Turtle's formula change means a Turtle player with 1 force in a region now loses that force (lost = max(1, 0) = 1, leaving 0), making them evictable.

Three residual concerns remain, none blocking: the legacy MANDATE handler on the server is dead code that could confuse future maintenance; the Dragonfly tie behavior is a design choice that could be intentional but is undocumented; and the skip-phase escape hatch on an empty draw (which the reshuffle guard makes nearly unreachable) skips to War rather than advancing to the next player's mandate turn.

## Legacy MANDATE handler creates unvalidated execution path

The server in `src/server/index.ts` still contains:

```typescript
case 'MANDATE': { const l=lobbies.get(currentLobbyId||''); if(!l?.gameState)return;
  let s=executeMandate(l.gameState,data.payload.mandate,data.playerId);
  s=advancePlayer(s); l.gameState=s; broadcastState(l); break; }
```

This executes any mandate type the client sends without checking `mandatesDeck`, without setting `mandateChoicePhase`, and without returning unused cards. A malicious or buggy client could send `{ type: 'MANDATE', payload: { mandate: 'betray' } }` repeatedly to steal coins without ever drawing from the deck. The new `CHOOSE_MANDATE` handler correctly calls `chooseMandateFromDrawn` which validates the chosen mandate is in `drawnMandates`, but the old handler has no such check. Removing the `MANDATE` case would close this path.

## Dragonfly honor scoring in online multiplayer

The duplicate-clan prevention only applies to hotseat mode's MainMenu. The online `JOIN_LOBBY` handler accepts whatever `clanId` the client sends with no uniqueness check. Two online players could both pick Dragonfly, and both would receive the honor VP bonus every round (since both start at 5 honor, both satisfy `p.honor >= maxHonor`). This is a minor concern because the bonus is only +1 VP per scored region, but it creates a cooperative dominant strategy for two colluding players.

## Skip-phase semantics

In `ActionPanel.tsx`, when `mandateChoicePhase` is true but `drawnMandates` is empty:

```typescript
<button className="btn-primary" onClick={doAdvancePhase}>Skip - Advance Phase</button>
```

`doAdvancePhase` calls `advancePhase` which transitions from politics to war. The correct behavior for "I can't draw" would be `advancePlayer` (move to next player's mandate turn, or advance phase if `politicsMandateCount >= maxMandates`). With the reshuffle guard producing a fresh 30-card deck whenever empty, this code path is essentially unreachable — but if it were reachable, it would skip all remaining players' mandates for the season.
