# Monster card effects and force calculation fixes (v2)

This change fixes three force-calculation bugs (Bishamon, Sacred Warrior, Phoenix in Betray) and implements 13 monster card effects spanning war-phase triggers, pre-battle effects, death triggers, on-enter effects, and movement restrictions. The v2 pass addressed the two most critical issues from v1: Earth Dragon and Jorogumo now fire regardless of step-by-step mode, and the Daikaiju condition was simplified.

**Watch for:** Daikaiju targeting condition is still tautological — `owners.length >= 1` is always true in a battle province, so it targets the first slot unconditionally rather than picking a contested province (confirmed). Koneko/Ebisu death triggers fire from `finalProvince` before figures are actually removed — Koneko penalizes the battle winner (who still has figures there) as "other players in province" (confirmed).

## High-level view

The Daikaiju "fix" replaced `owners.length >= 2 || owners.length >= 1` with just `owners.length >= 1`, which is still always true for any province in `battleProvinceIds`. The monster targets the first battle province in slot order, ignoring contestedness.

Earth Dragon and Jorogumo are now applied unconditionally in `resolveNextBattle`, outside the `!stepByStepMode` gate. The online-mode gap from v1 is closed.

Koneko's death trigger uses `province.figures` to identify "other players in province." When Koneko dies as a battle casualty, `finalProvince` still contains the winner's figures (removal happens a few lines later). The winner gets penalized with -2 coins and -2 ronin for killing Koneko. Whether this matches the card's intent depends on interpretation — the winner is indeed present — but it punishes the victor for winning.

## Daikaiju targeting condition

The v1 review flagged `owners.length >= 2 || owners.length >= 1` as tautological. The fix removed the first disjunct, leaving:

```typescript
const targetBattleProvince = battleProvinceIds.find(bpId => {
  const bp = newState.provinces[bpId];
  if (!bp) return false;
  const owners = [...new Set(bp.figures.map(f => f.owner))];
  return owners.length >= 1;
});
```

Every province in `battleProvinceIds` has at least one figure. The condition is still always true. Daikaiju moves to the first battle province in slot order regardless of whether multiple players are present. If the intent is "place Daikaiju in a contested battle," the threshold should be `>= 2`.

## Koneko death trigger timing in battle

```typescript
battle.participants.forEach((pid) => {
  if (pid === winnerId) return;
  if (winner.allies.includes(pid)) return;
  const loserFiguresForTriggers = finalProvince.figures.filter(f => f.owner === pid && f.type !== 'fortress');
  for (const fig of loserFiguresForTriggers) {
    if (fig.type === 'monster' && fig.monsterCardId === 'su-koneko') {
      applyKonekoDeathEffect(newState, battle.provinceId, pid);
    }
  }
});
```

`applyKonekoDeathEffect` reads `state.provinces[provinceId].figures` — which at this point is still `finalProvince` (the removal happens after this block). The winner's figures are present, so `otherPlayersInProvince` includes the winner. The winner loses 2 coins and 2 ronin for killing Koneko.

## Bishamon force semantics

The implementation checks for ANY other monster in the province:

```typescript
const hasOtherMonster = province.figures.some(
  f => f.type === 'monster' && f.monsterCardId !== 'su-bishamon' && f.id !== fig.id
);
```

The user specified "si hay otro monstruo en la misma provincia" (any other monster), which the code follows. The printed card text says "Force 4 while opponent Monster in same Province," which would restrict the check to `f.owner !== fig.owner`. The discrepancy between user spec and card text should be confirmed.
