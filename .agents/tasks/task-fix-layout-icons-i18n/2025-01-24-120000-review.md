# CSS layout fixes and Spanish log translations

Two CSS one-liners fix the Shinto button overflow and the premature icon row-wrapping on region cards. Separately, ~76 hardcoded English log messages in `gameLogic.ts` and `gameStore.ts` are replaced with Spanish equivalents. The translation pass covers the game log thoroughly but does not touch card power descriptions or kami effect texts, which remain in English despite the game being configured in Spanish.

**Watch for:** Card/kami power descriptions still display in English (confirmed) — the user explicitly requested these translated. Some translated strings use ASCII approximations for accented characters ("Otono" instead of "Otoño", "Te" instead of "Té") which may look odd in-game (confirmed).

## High-level view

The `flex-wrap:wrap` addition to `.recruit-buttons` is the minimal correct fix for the Shinto button overflow. The `max-width` bump from `calc(5 * 23px)` to `calc(5 * 24px)` in `.region-forces` gives 120px of row width against the 117px needed (5 icons × 21px + 4 gaps × 3px), though the magic number "24" doesn't correspond to any actual dimension and could be expressed more clearly as `calc(5 * 21px + 4 * 3px)`.

The translation effort covers every `log` push in both files — season setup, mandates, war, kami resolution, cleanup, winter scoring. However, the `effect` field on every `SeasonCard` and every `KamiData` entry in `types/game.ts` remains in English and is rendered directly in SeasonCardsMarket, SeasonCardsModal, PlayerCardsModal, RegionDetailModal, and MainMenu. This is the "card power description" the user called out.

Several translated strings omit diacritical marks: "Estacion" (Estación), "Otono" (Otoño), "Te" (Té), "Politica" (Política), "ningun" (ningún). These are cosmetic but noticeable in a Spanish UI, especially since the existing `es.ts` i18n file uses proper UTF-8 characters.

## Untranslated card and kami descriptions

The user's request explicitly mentions "las cartas la descripcion del poder sale en ingles." The `effect` field on `SeasonCard` is a plain string defined in `src/types/game.ts` and rendered directly:

```tsx
// SeasonCardsMarket.tsx
<div className="card-effect">{card.effect}</div>

// SeasonCardsModal.tsx
<p className="season-card-effect">{card.effect}</p>

// PlayerCardsModal.tsx
<p className="season-card-effect">{card.effect}</p>
```

There are approximately 60+ season cards across SPRING_CARDS, SUMMER_CARDS, and AUTUMN_CARDS, all with English `effect` text like:

```typescript
{ id: 'sp-benevolence', name: 'Benevolence', cost: 1, ..., effect: 'When you spend Coins, you may give one to another player. If you do, gain Honor and 2 VP.' }
```

Similarly, `KAMI_DATA` has 5 entries with English `effect` strings:

```typescript
{ type: 'amaterasu', name: 'Amaterasu', effect: 'Allows worshiper to move clan marker to top of Honor Track.' }
```

These are rendered in the MainMenu kami selector and as tooltips throughout the game UI.

## Missing diacritics

Throughout `gameLogic.ts` and `gameStore.ts`, accented Spanish characters are approximated with ASCII:

- "Preparacion de Estacion" → "Preparación de Estación"
- "Otono" → "Otoño"
- "Ceremonia del Te" → "Ceremonia del Té"
- "Comienza la Fase de Politica" → "Política"
- "ningun santuario" → "ningún santuario"
- "rehen(es)" → "rehén(es)"
- "Puntuacion Final" → "Puntuación Final"
- "sin oposicion" → "sin oposición"

The existing `es.ts` i18n file uses proper UTF-8 throughout, so this is an oversight in the translation pass rather than a technical constraint.

## Opaque max-width formula

The formula `calc(5 * 24px)` gives 120px against the 117px needed (5 × 21px icons + 4 × 3px gaps). The number "24" doesn't correspond to any CSS dimension in the codebase — it's `icon_size + gap` rounded up, which isn't obvious. Expressing it as `calc(5 * 21px + 4 * 3px)` would survive changes to icon size or gap without requiring someone to reverse-engineer the arithmetic.
